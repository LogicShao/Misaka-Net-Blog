# Electron 博客管理后台使用指南

## 🚀 快速开始

### 方法 1：使用启动脚本（推荐）

双击项目根目录下的 `启动博客管理后台.bat`，应用会自动启动。

### 方法 2：命令行启动

```bash
# 进入应用目录
cd electron-admin

# 启动应用
npm start
```

## 📋 功能说明

### 1. 文章管理

- **新建文章**：点击顶部"➕ 新建文章"按钮或按 `Ctrl+N`
  - 自动生成时间戳格式的文件名（如 `25-11-22-14-30.md`）
  - 填写标题、描述、日期、标签
  - 编写 Markdown 内容
  - 点击"✨ 创建文章"保存

- **编辑文章**：
  - 在左侧列表中点击任意文章
  - 在右侧编辑器中修改内容
  - 支持三个标签页：
    - **元数据**：编辑标题、描述、日期、标签、草稿状态
    - **内容**：编辑 Markdown 正文
    - **预览**：查看原始数据
  - 点击"💾 保存更改"

- **删除文章**：
  - 选中文章后点击"🗑️ 删除文章"
  - 确认后删除（不可恢复）

### 2. 搜索功能

- 在左侧搜索框中输入关键词
- 支持按**标题**和**标签**搜索
- 实时过滤结果

### 3. 统计信息

左侧顶部显示：
- **总计**：所有文章数量
- **已发布**：非草稿文章数量
- **草稿**：标记为草稿的文章数量

### 4. 构建博客

点击顶部"🔨 构建博客"按钮或按 `Ctrl+B`：
- 弹出构建进度窗口
- 显示构建日志
- 构建完成后可在 `dist/` 目录查看结果

### 5. 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+N` | 新建文章 |
| `Ctrl+B` | 构建博客 |
| `Ctrl+R` | 刷新列表 |
| `F11` | 全屏 |
| `F12` | 开发者工具 |

## 💡 使用技巧

### 文章命名规范

建议使用时间戳格式：`YY-MM-DD-HH-MM.md`（应用会自动生成）

### 标签管理

- 多个标签用**逗号分隔**
- 示例：`技术, 教程, JavaScript`
- 标签会自动去除首尾空格

### 草稿功能

- 勾选"标记为草稿"后，文章会：
  - 在列表中显示橙色"草稿"徽章
  - 在生产环境构建时自动过滤
  - 仅在开发环境显示

### 日期格式

- 使用日期选择器选择发布日期
- 格式：`YYYY-MM-DD`（如 `2025-11-22`）
- 会影响文章在博客中的排序

## 🛠️ 打包应用

如果想将应用打包成独立的可执行文件：

```bash
cd electron-admin

# Windows 应用
npm run build:win

# macOS 应用
npm run build:mac

# Linux 应用
npm run build:linux
```

打包后的文件在 `electron-admin/dist/` 目录。

## ❓ 常见问题

### Q: 应用无法启动

A:
1. 确保已安装依赖：`cd electron-admin && npm install`
2. 检查 Node.js 版本 >= 18
3. 查看错误日志

### Q: 找不到文章

A: 确认文章位于 `src/content/blog/` 目录

### Q: 构建失败

A:
1. 检查 `package.json` 中是否有 `build` 脚本
2. 手动运行 `npm run build` 测试
3. 查看构建日志中的错误信息

### Q: 修改后不生效

A: 点击"🔄 刷新"按钮或按 `Ctrl+R`
